## ▌ 0. 系統總體哲學與邊界限制
戰略構想: 「免 API 替代方案」！三條極具彈性與戰略價值的「非官方 / 游擊隊」路線。CEO提到的這三個替代方案，在我們「預算 <=US$100/month」的極限條件下，這些方案確實是救命解法。：

🎯 1. 繞道戰術：無頭瀏覽器自動化 (Browser Automation)
CEO的洞見：利用 dream-to-video-skill 的 Playwright 技術，不走 API，直接操控網頁端的高級帳號。
對產線的意義：這招是真正的「成本殺手」。我們只要花 $70 買一個網頁版即夢 Premium，就能讓 $$VD (視覺導演) 透過 Playwright 腳本在背景自動登入、填字、下載。優點是徹底省下高昂的 API Token 費用；缺點是只要即夢官網改版，腳本就會罷工，需要工程團隊隨時修補 DOM 節點。

🎯 2. 核心大腦：純提示詞生成 (Prompt Optimization)
CEO的洞見：利用 Seedance2.0-Prompt-Optimizer-skill 的邏輯，專注於生成完美符合 Seedance 2.0 審查與視覺特性的提示詞，甚至可以手動貼上。
對產線的意義：這正是我們已經在做的事！我們的 $$CW (編劇) 透過 batch_chimera_blueprint_factory.py 生成的 JSON，其實就是吸收了這種「提示詞最佳化」的精華。它確保了我們產出的 Prompt 帶有 1990s stop-motion style 等完美繞過審查且維持美學的魔法指令。

🎯 3. 戰術兜底：網頁資源抓取 (Web Scraping / Downloader)
CEO的的洞見：像 jimeng_downloader 這樣的工具，透過 DOM 提取無浮水印的原始畫質檔案。
對產線的意義：如果我們真的採用上述的 Playwright 網頁自動化路線，這個 DOM 提取技術就是我們 $$QA (剪輯師) 自動把完成的影片從網頁端「拔」下來、送進 FFmpeg 的關鍵技術兜底。

* **JSON 暴力清洗鐵律 (Robust Parsing)**：所有接收大模型回傳 JSON 的腳本，必須實作正則表達式 (Regex) 剝除邏輯，過濾掉 LLM 幻覺產生的 ` ```json ` 等 Markdown 標籤，防止解析崩潰 (`JSONDecodeError`)。

## ▌ 2. 全新防禦陣線：前端哨兵系統 (The DOM Sentinel)
目標：即時探測即夢網頁改版，保護高級帳號不被反爬蟲機制封鎖。
* **運作機制與本地壓縮降級 (Auto-Diagnostic)**：
  1. **日常巡邏**：在 `$$VD` 啟動前強制執行。透過 Headless Chromium 探測即夢頁面的關鍵 CSS Selectors。若正常，則放行產線。
  2. **Cookie 憑證續期防護**：若偵測到「請登入」畫面，立即中斷 (`sys.exit(1)`) 並觸發 Telegram 警報：「🚨 帳號 Cookie 已過期，請重新登入！」
  3. **改版防呆、視覺快照與本地去噪 ($0 成本備援)**：若發現 DOM 改版，哨兵在 `sys.exit(1)` 之前，必須執行兩件事：
     * **視覺快照 (Visual Anchor)**：強制 Playwright 拍下全螢幕截圖 (`screenshot.png`)。
     * **本地去噪**：將網頁原始碼交給 `BeautifulSoup` 剝除無用 `<script>`/`<style>`，輸出為純淨結構樹 (`dom_error_snapshot.txt`)。
     週末復盤時，結合截圖與乾淨 DOM，能讓 LLM 100% 精準定位新按鈕。
  4. **記憶體洩漏防禦 (Hard Close)**：無論是哨兵還是 RPA 算圖引擎，每次任務結束或發生異常時，必須強制呼叫 `context.close()` 與 `browser.close()` 徹底銷毀執行緒，嚴禁在 24/7 運行中重複使用舊 Context，防止 Mac mini M4 發生 OOM (記憶體溢出) 崩潰。

## ▌ 3. 跨齒輪資料交接橋樑 (Data Handoff Protocol)
**目標：確保齒輪一與齒輪二在 24/7 非同步運作時，不會發生資源衝突與死鎖。**
* **唯一交集點**：齒輪二 (永動寫入端) 與齒輪一 (產線讀取端) 唯一的交集點是 SQLite 基因金庫 `assets/data/style_vault.db`。
* **併發防護機制 (Lock Prevention)**：底層 `style_database.py` 已強制啟用 `PRAGMA journal_mode=WAL;` (Write-Ahead Logging 模式)。團隊在撰寫任何資料庫連線時，必須強制配置 `timeout=30` 參數。這能確保兩套系統在極限併發讀寫時，能在底層安全排隊，徹底根除 `Database Locked` 致命錯誤。

### 🔧 4.0 [需新增] 跨平台動態環境中樞 (`env_manager.py`)
**狀態**：目前不存在，需由 RD 團隊全新設計。
**說明** : 1. 抽離「絕對路徑」，改用 .env 與 pathlib
Windows 和 macOS 最致命的差異就是路徑格式（F:\ vs /Volumes/ 以及 \ vs /）。
規範：全面禁用硬編碼 (Hardcoding) 路徑。請工程團隊建立一個統一的 .env 檔案來切換環境。

在 Windows 開發時的 .env：
# 系統環境切換 (WINDOWS / MAC_M4)
TARGET_ENV=WINDOWS
# Windows 工作區路徑
WORKSPACE_ROOT=F:\AI_DRAMA_Factory
在 4/18 遷移到 Mac 時的 .env：
TARGET_ENV=MAC_M4
# Mac 外接 APFS 硬碟路徑
WORKSPACE_ROOT=/Volumes/AI_Workspace/AI_DRAMA_FACTORY
2. 建立全域環境偵測模組 (env_manager.py)
請工程團隊新增這支核心的配置腳本。未來的 auto_editor.py 或 batch_style_analyzer_sqlite.py 只要 import 這支腳本，系統就會「自動意識」到自己現在是跑在 Windows 還是 Mac 上。

"""
🏭 齒輪一 (生產部) - 前端哨兵系統 (The DOM Sentinel)
檔案名稱: jimeng_dom_sentinel.py
核心功能: 
1. 探測即夢 (Jimeng) 網頁 DOM 是否改版
2. 偵測 Cookie 是否過期 (Session Decay)
3. 若發生異常，使用 BeautifulSoup 在本地端進行 HTML 去噪壓縮 ($0 成本防呆)
4. 將報錯寫入 project_learning.md (啞巴版復盤)

def send_telegram_alert(message: str):
    """發送緊急 Telegram 警報"""
    bot_token = os.getenv('TELEGRAM_BOT_TOKEN')
    chat_id = os.getenv('TELEGRAM_CHAT_ID')
    if bot_token and chat_id:
        try:
            import requests
            url = f"https://api.telegram.org/bot{bot_token}/sendMessage"
            requests.post(url, json={"chat_id": chat_id, "text": message}, timeout=10)
        except Exception as e:
            print(f"⚠️ Telegram 警報發送失敗: {e}")

def log_fatal_error(error_code: str, details: str):
    """【CTO 鐵律】啞巴版復盤：將錯誤寫入 project_learning.md 並強制退出"""
    timestamp = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    log_entry = f"\n### [{timestamp}] DOM Sentinel Error: {error_code}\n* **Details**: {details}\n* **Action**: Auto-halted pipeline.\n"
    
    try:
        with open(LEARNING_LOG_FILE, "a", encoding="utf-8") as f:
            f.write(log_entry)
        print(f"📝 錯誤已自動 Append 至 {LEARNING_LOG_FILE.name}")
    except Exception as e:
        print(f"⚠️ 無法寫入學習日誌: {e}")
        
    print(f"❌ 產線已強制中斷 (Exit Code 1): {error_code}")
    sys.exit(1)

def compress_and_save_dom(html_content: str):
    """【CTO 鐵律】本地降級壓縮：用 BeautifulSoup 剝除無用標籤 ($0 成本)"""
    print("🧹 啟動 BeautifulSoup 本地 HTML 壓縮降級去噪...")
    soup = BeautifulSoup(html_content, "html.parser")
    
    # 暴力剝除所有 script, style, svg, path 等干擾大模型視覺的標籤
    for tag in soup(["script", "style", "svg", "path", "img", "meta", "link", "iframe"]):
        tag.decompose()
        
    # 提取純淨的結構化文字與核心按鈕/輸入框屬性
    clean_text = soup.get_text(separator='\n', strip=True)
    
    try:
        DOM_SNAPSHOT_FILE.parent.mkdir(parents=True, exist_ok=True)
        with open(DOM_SNAPSHOT_FILE, "w", encoding="utf-8") as f:
            f.write("=== CLEANED DOM SNAPSHOT ===\n")
            f.write(clean_text)
        print(f"💾 去噪後的 DOM 快照已存至: {DOM_SNAPSHOT_FILE.name} (檔案大幅縮小，適合餵給低成本 LLM 分析)")
    except Exception as e:
        print(f"⚠️ 儲存 DOM 快照失敗: {e}")

            # 尋找畫面上是否有明顯的「登入」按鈕，如果出現代表被踢出了
            if page.locator(SELECTOR_LOGIN_BTN).count() > 0:
                msg = "🚨 【即夢警報】帳號 Cookie 已過期！產線已強制暫停，請立即於本機執行 `setup_jimeng_session.py` 重新登入！"
                send_telegram_alert(msg)
                browser.close()
                log_fatal_error("SESSION_DECAY", "Detected login button. Cookie is invalid or expired.")

            # --- 偵測 2：網頁改版 (DOM 結構變更) ---
            print("🔍 掃描關鍵 DOM 節點 (輸入框與生成按鈕)...")
            try:

                page.wait_for_selector(SELECTOR_GENERATE_BTN, timeout=10000)
                print("✅ DOM 結構健康！輸入框與生成按鈕皆存在。")
                
            except PlaywrightTimeoutError:
                # 發生改版！找不到節點
                msg = "🚨 【即夢警報】網頁 DOM 結構已改變！前端哨兵已攔截異常並啟動本地去噪壓縮，請工程師查看 `dom_error_snapshot.txt`。"
                print("❌ 警告：找不到關鍵 DOM 節點，疑似網頁改版！")
                send_telegram_alert(msg)
                
                # 取得當下崩潰的網頁原始碼，交給 BeautifulSoup 處理
                raw_html = page.content()
                compress_and_save_dom(raw_html)
                
                browser.close()
                log_fatal_error("DOM_CHANGED", f"Failed to find {SELECTOR_PROMPT_INPUT} or {SELECTOR_GENERATE_BTN}")

        except Exception as e:
            # 網路斷線或其他未知錯誤
            msg = f"🚨 【即夢警報】前端哨兵發生未知崩潰: {str(e)[:100]}"
            send_telegram_alert(msg)
            browser.close()
            log_fatal_error("UNKNOWN_CRASH", str(e))
            
        # 安全關閉
        browser.close()
        
    print("✅ 前端哨兵探測完畢，放行產線！(Exit Code 0)")
    sys.exit(0)

if __name__ == "__main__":
    main()
