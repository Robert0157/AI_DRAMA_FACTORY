# 系統總體架構白皮書：提線社畜 (游擊隊 RPA v7.9 終極防護版)
**發布日期**: 2026-03-25 | **核心戰略**: 零 API 成本、無頭瀏覽器 RPA、DOM 即時監控
**目標環境**: Mac mini M4 (Apple Silicon) + 外接 APFS 硬碟 (/Volumes/AI_Workspace/)

## ▌ 0. 系統總體哲學與邊界限制
本系統為極限壓縮成本的 B2B 商業微電影產線。視覺生成全面棄用官方 API，改採 Playwright 操控即夢 (Jimeng) 高級網頁版。在「預算 <=US$100/month」的極限條件下，我們必須嚴格遵守以下戰略與鐵律：

### 🎯 核心游擊戰略 (The Guerilla Tactics)
1. **繞道戰術 (CDP 劫持自動化)**：利用 `subprocess` 獨立啟動真實 Chrome 讓用戶完成人類登入，事後再用 Playwright 的 `connect_over_cdp` 接管 Session。100% 繞過大廠 SSO 防護網，實現零 API 成本生成。
2. **流程戰術 (Music First 與動態踩點)**：視覺預算鎖死於每月即夢 Premium 訂閱費。音樂強制先行且純器樂，利用 `librosa` 抓取重拍時間戳，再交由 LLM 生成對應分鏡，視覺強制植入「定格動畫、木偶」等關鍵字以降低 AI 生成瑕疵。
3. **封裝戰術 (開源 UI 攔截器)**：整合 `chatgpt-web-midjourney-proxy` 作為導演中控台。在本地架設 FastAPI 攔截請求，將網頁點擊轉化為背後 RPA 礦車的操作，前端華麗接單，後端零成本白嫖。

### 🛡️ 產線防禦與基礎設施鐵律 (The Ironclad Rules)
* **物理路徑鎖死**：保護 Mac M4 的 256GB SSD，所有大容量 I/O (影片下載、截圖、資料庫) 強制綁定 `/Volumes/AI_Workspace/`。 (4/18/2026 開始生效)
* **DOM 監控鐵律**：部署「前端哨兵」即時監控即夢網頁改版，做到「發現改版 -> 立即停機 -> 人工修復」。
* **標準化退出碼與日誌瘦身 (Log Sanitization)**：所有 Python 腳本在捕捉到致命錯誤時，必須強制呼叫 `sys.exit(1)`。寫入 `project_learning.md` 的錯誤日誌必須經過正則清洗，嚴禁寫入動輒數十行的 Stack Trace，僅保留「Exception Name」與「最後一行關鍵錯誤訊息」，防止檔案無限膨脹導致復盤 Token 暴增。
* **指令注入防禦 (Command Injection Prevention)**：在 `auto_editor.py` 或任何呼叫 FFmpeg 等底層命令的腳本中，嚴禁使用 `shell=True`。所有 Subprocess 必須以陣列形式傳遞參數 `["ffmpeg", "-i", ...]`，並對 LLM 生成的檔名/路徑強制執行安全字元過濾，防止惡意 Prompt 注入導致 `rm -rf /` 災難。
* **原生外部機密隔離 (External Secrets)**：徹底淘汰實體 `.env` 檔案。全面導入 OpenClaw 的「外部機密管理」，透過終端機執行 `openclaw secrets configure` 寫入 API 金鑰。腳本運行時透過 `target-path` 驗證動態調用，實現「零明文落地」的軍規級防護。
* **API 級別結構化輸出 (Structured Outputs)**：捨棄易脆斷的正則表達式清洗。所有呼叫大模型的腳本，必須在 API 請求層面強制啟用 `response_format={ "type": "json_object" }` 或給定嚴格 JSON Schema。從底層強制約束 100% 合法 JSON，將防呆成本降至 $0。
* **原子寫入防護 (Atomic Writes)**：針對憑證檔 (`state.json`) 與狀態池，嚴禁直接 `open(file, 'w')`。必須強制寫入 `.tmp` 暫存檔，寫入成功後再呼叫作業系統層級的 `os.replace()` 覆蓋原檔。確保在斷電或磁碟瞬斷時，JSON 檔案 100% 不會發生 0KB 靜默損毀。
* **用完即殺與殭屍進程清剿 (Kill-on-Yield & Zombie Hunter)**：Meta-Agent 在收到 `sessions_yield` 時，必須強制呼叫 `/compact` 搭配 `hard_clear` 策略，粉碎上下文記憶體。腳本層面交接前，必須強制呼叫 `os.system("pkill -9 -f Chromium")` 與 `pkill -9 -f ffmpeg` 無情獵殺殘留程序，徹底根絕 Mac M4 統一記憶體的 OOM (Out of Memory) 崩潰。---

## ▌ 2. 全新防禦陣線：前端哨兵系統 (The DOM Sentinel)
目標：即時探測即夢網頁改版，保護高級帳號不被反爬蟲機制封鎖。
* **運作機制與本地壓縮降級 (Auto-Diagnostic)**：
  1. **日常巡邏**：在 `$$VD` 啟動前強制執行。透過 Headless Chromium 探測即夢頁面的關鍵 CSS Selectors。若正常，則放行產線。
  2. **無感 Cookie 輪替與持久化 (Session State)**：哨兵不再只會報錯停機。若偵測到「請登入」等憑證失效畫面，哨兵將暫停產線，但在背景自動喚醒「備用登入腳本」：載入 `jimeng_auth_state.json` 或利用預存金鑰模擬無頭登入刷新 Token。刷新成功後自動接續產線，實現真正的 24/7 無人值守。
  3. **改版防呆、視覺快照與本地去噪 ($0 成本備援)**：若發現 DOM 改版，哨兵在 `sys.exit(1)` 之前，必須執行兩件事：
     * **視覺快照 (Visual Anchor)**：強制 Playwright 拍下全螢幕截圖 (`screenshot.png`)。
     * **本地去噪**：將網頁原始碼交給 `BeautifulSoup` 剝除無用 `<script>`/`<style>`，輸出為純淨結構樹 (`dom_error_snapshot.txt`)。
     週末復盤時，結合截圖與乾淨 DOM，能讓 LLM 100% 精準定位新按鈕。
  4. **記憶體洩漏防禦與互斥鎖 (Hard Close & Mutex)**：無論是哨兵還是 RPA 算圖引擎，任務結束時必須強制執行 `browser.close()` 與 `playwright.stop()` 徹底銷毀程序。交接給 `$$QA` (FFmpeg) 之前，必須執行系統級記憶體檢查，確保 Chromium 進程已被完全釋放，防範 M4 發生瞬間凍結。

## ▌ 3. 跨齒輪資料交接橋樑 (Data Handoff Protocol)
**目標：確保齒輪一與齒輪二在 24/7 非同步運作時，不會發生資源衝突與死鎖。**
* **唯一交集點**：齒輪二 (永動寫入端) 與齒輪一 (產線讀取端) 唯一的交集點是 SQLite 基因金庫 `assets/data/style_vault.db`。
* **併發防護機制 (Lock Prevention)**：底層 `style_database.py` 已強制啟用 `PRAGMA journal_mode=WAL;` (Write-Ahead Logging 模式)。團隊在撰寫任何資料庫連線時，必須強制配置 `timeout=30` 參數。這能確保兩套系統在極限併發讀寫時，能在底層安全排隊，徹底根除 `Database Locked` 致命錯誤。

## ▌ 4. 齒輪二：R&D 逆向研發線 (Gear 2 - 永動機)
**運作環境**: 獨立背景執行緒 (24/7 常駐)  
* **排程機制升級 (Cron/Launchd)**：全面廢除容易導致 OOM (記憶體溢出) 與無聲死亡的 `while True` 無限迴圈腳本 (`infinite_miner_daemon.py`)。
