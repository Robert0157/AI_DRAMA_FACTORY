# 系統總體架構白皮書：提線社畜 (企業雙核心旗艦版 v6.0)
**發布日期**: 2026-03-25 | **系統核心**: OpenClaw 框架 + 雙軌原生 Python 引擎
**目標環境**: Mac mini M4 (Apple Silicon) + 外部 APFS 硬碟 (`/Volumes/AI_Workspace/`)

## ▌ 0. 系統總體哲學與邊界限制
本系統為全自動 B2B 商業微電影產線。架構分為「非同步運作」的兩大齒輪，嚴禁互相阻塞。
* **物理路徑鎖死**：保護 Mac M4 的 256GB SSD，所有大容量 I/O (影片下載、截圖、資料庫) 強制綁定 `/Volumes/AI_Workspace/`。
* **無人聲純器樂與版權紅線**：嚴禁爬取串流平台。配樂強制注入 `strictly instrumental, NO vocals`，並由 FFmpeg 執行 `-an` 靜音剝奪。
* **視覺瑕疵美學 (Aesthetic Lock)**：Prompt 強制植入「定格動畫、木偶、12fps 卡頓感」。

---

## ▌ 1. 跨齒輪資料交接橋樑 (Data Handoff Protocol)
齒輪二 (寫入) 與齒輪一 (讀取) 唯一的交集點是 `assets/data/style_vault.db`。
* **併發防護機制**：`style_database.py` 已強制啟用 `PRAGMA journal_mode=WAL;`。所有資料庫連線必須配置 `timeout=30`，確保兩套系統 24/7 同時運作時，絕不發生 `Database Locked`。

---

## ▌ 2. 齒輪二：R&D 逆向研發線 (Gear 2 - 永動機)
**運作環境**: 獨立背景執行緒 | **入口點**: `infinite_miner_daemon.py`
**核心目標**: 24/7 不間斷為 SQLite 金庫注入神級美學基因。

永動機由 `while True` 驅動，包含以下三大精密模組：

### 2.1 詞彙變異器 (`daily_keyword_mutator.py`)
* **煉金術**：呼叫 `glm-4-plus`，將極端美學與社畜痛點融合，每日產出 30-50 個帶有 `Lofi Type Beat` 或 `MV` 後綴的冷門關鍵字，寫入 `video_input.txt`。

### 2.2 核心挖礦引擎 (`batch_style_analyzer_sqlite.py` - 三合一重構版)
這是齒輪二的重裝裝甲車，具備以下企業級特性：
* **智能斷點續傳 (Smart Resume)**：啟動時優先讀取 `.url_pool.json` 快取，並比對 `processing_log` 資料表。自動跳過已 `SUCCESS` 與 `FAILED` 的 URL，達成 0 Token 浪費。
* **3-Worker 並行加速**：使用 `ThreadPoolExecutor` 開啟 3 條執行緒同時下載與分析影片，處理吞吐量提升 300%。
* **視覺駭客戰術 (Contact Sheet Hack)**：利用 `Pillow` 將 `ffmpeg` 提取的 12 張關鍵幀，無縫拼接成一張 `384x216` 解析度的「十二宮格縮圖大圖」，完美繞過 API 多圖負載報錯。
* **多模態提煉**：將縮圖大圖送入 `glm-4v-plus`，精準剝離出 `video_genes` 與 `audio_genes`，並寫入資料庫。

### 2.3 庫存管家 (`vault_manager.py`)
* **維運工具**：提供 CLI 介面供工程師進行資料庫的 `stats`, `list`, `search`, `export` 與清理，確保基因庫的健康度。

## ▌ 3. 齒輪一：日常自動化產線 (Gear 1 - OpenClaw)
**運作環境**: OpenClaw Framework | **入口點**: Meta-Agent `project.yml`

捨棄外部不穩定的開源套件，全面採用團隊自研的 **Local Skills (原生 Python 腳本)** 進行產線組裝：

### 3.1 [Protocol PM-Init] 合成獸藍圖工廠 (`batch_chimera_blueprint_factory.py`)
* `$$PM` 從資料庫隨機抽出 3 筆無關聯的基因。由 `$$CW` (呼叫 `glm-4-plus`) 進行基因縫合，產出帶有多語系旁白與混合美學的 `blueprint_xxx.json`。

### 3.2 [Protocol A] 跨境音效套利引擎 (`chinese_api_audio_engine.py`)
* **Music-First 原則**：`$$AD` 優先呼叫低成本 API 進行算力套利。腳本強制綁架 Prompt 確保全球化純器樂，並輸出至 APFS 硬碟。隨後由 `ffprobe` 測量精準長度，回傳給 `$$CW`。

### 3.3 [Protocol S] 分鏡切割與 Telegram 審批 (`telegram_approver_audio.py`)
* `$$CW` 依據音軌精確秒數，切割出至多 15 鏡的 JSON 劇本。
* `$$PM` 呼叫 Telegram 腳本，啟動 `bot.polling(timeout=300)` 執行緒阻塞。等待老闆點擊【✅ 批准】後，才放行至高成本的算圖階段。

### 3.4 [Protocol V & Q] M4 硬體加速剪輯台 (`auto_editor.py`)
* `$$VD` 完成即夢算圖後，交由 `$$QA` 呼叫此腳本。
* **極致壓榨**：啟用 Apple Silicon 原生硬體編碼器 `-c:v h264_videotoolbox`。若發生異常，自動降級為 CPU 軟解 (`libx264`) 防崩潰。
* 腳本自動處理單引號脫逸防錯，將無聲影像與套利配樂合併，壓製 macOS 原生字體 (`PingFang.ttc`) 字幕，最後產出 `receipt.json` 版權護身符。

### 3.5 [Protocol D] 財務核算與垃圾回收 (`ledger_manager.py` / `cloud_archiver.py`)
* 每次生成完畢，寫入 `budget_ledger.db` 核算 CPV。
* 將成品打包 ZIP 預備上傳 S3。**目前維持 `Garbage Collection SKIPPED` 狀態**，待冷儲存串接驗證無誤後再開啟本地端 `rm -rf`，保障資料安全。
