**核心戰略升級**：
全面升級為 v15.3 終極穩定架構。導入 **Streamlit 單一中控樞紐**（徹底廢除終端機 CLI 操作）、**Encode-Once-Repeat 極速渲染**（75 倍速物理拼接，捨棄全片重壓）、**Pre-baked Fade 呼吸燈轉場**（無損實現 Dip to Black）、**Acrossfade 多音軌預混音**（徹底解決 FFmpeg 碎檔時間軸漂移），以及 **全域 Gemini 2.5 Flash 統一供彈**（消滅智譜 GLM-4）。系統現已達到 100% 商業運營、極速產出與零 UI 阻塞的完美狀態。

> **The Cost & Speed Optimized Strategy (成本與極速最佳化戰略)**：  
> 產線由「高耗能渲染」轉向「模組化物理組裝」。透過預先將 8 秒微動態影片鑄造為帶有呼吸燈效果的基底 (in/mid/out)，再使用 `concat demuxer` 無損複製延長，1 小時影片的視覺渲染時間從 2 小時暴降至 5~10 分鐘以內。大腦全面切換至 Gemini 2.5 Flash，藉由非同步背景執行緒 (Background Threading)，CEO 可在 Web UI 絲滑操作，將 API 成本與等待時間壓縮至最低極限。

### 🛡️ 產線防禦與基礎設施鐵律 (The Ironclad Rules)
* **Streamlit 樞紐單一化 (UI Singularity)**：廢除 `rs_manager.py` 與 `video_processor.py`。所有系統操作、幻影輪播、母帶回收、金庫代謝與發行審計，必須 100% 透過 `Streamlit` Web UI 進行。
* **背景執行緒防堵 (UI Non-Blocking)**：任何耗時任務（API 呼叫、FFmpeg 算圖）若從 Web UI 觸發，必須使用 Python `threading.Thread(daemon=True)` 在背景執行，嚴禁卡死主網頁介面。
* **絕對音畫同步與防漂移 (Absolute A/V Sync)**：廢除 `-shortest` 旗標以防時間戳漂移。音軌必須先在 Stage 2.5 預混音為單一超長 `.wav`，影像則在 Stage 2 強制以 `-t 3600` 鎖死精確時長，確保不多一秒、不少一秒。
* **雙棲實體路徑隔離**：透過 `.env` 檔案中的 `WORKSPACE_ROOT` 變數與 `env_manager.py` 進行跨平台路徑解析。Windows 開發期設定為本地目錄，Mac 部署後強制綁定指定路徑。
* **標準化退出碼與單行進度條 (Console Sanitization)**：核心錯誤必須呼叫 `sys.exit(1)`。FFmpeg 執行時嚴禁使用 `capture_output=True` 致盲，必須使用 `Popen` 攔截 `stderr` 轉化為 `\r` 單行進度條。
* **全域 Gemini 結構化輸出 (Structured Outputs)**：捨棄 GLM-4。所有企劃與提示詞生成必須使用 `google.generativeai` SDK 呼叫 Gemini 2.5 Flash，並給定嚴格 JSON Schema 確保 100% 合法輸出。
* **原子寫入防護 (Atomic Writes)**：針對 JSON 中繼資料與資料庫狀態，必須強制寫入 `.tmp` 暫存檔後再 `os.replace()`，防範斷電損毀。
* **最高修憲權限鎖定 (Constitution Immutability)**：嚴禁工程團隊、AI 輔助工具擅自修改產線憲法！核心文件更新僅限 CEO 或 CTO 手動執行。
* **零浪費與延遲扣款鐵律 (Zero Waste & Delayed Commit)**：視覺素材在抽樣階段【絕對禁止提前扣款】。必須在 1 小時成片成功寫入磁碟後，才針對中選的素材執行資料庫 `derivation_count + 1` 回寫。
* **多頻道物理隔離鐵律 (Channel Matrix Isolation)**：系統全線引入 `--channel` 參數，Lofi 與 Light Music 不同頻道的基因、資料庫與檔案必須 100% 隔離，嚴禁交叉污染。

## ▌ A. 交付物定義（驗收標準）
每次成功透過 Streamlit 啟動產線，系統必須輸出以下檔案至 `assets/final_exports/`，方可視為驗收通過：

1. **`R&S_Echoes_{channel}_1HrMix_[時間戳].mp4`**
   - 供 YouTube 上傳的 1 小時精確動態無縫影片。
   - 具備 Encode-Once-Repeat 視覺延伸、Pre-baked Fade 呼吸轉場與 100% 音畫同步。

2. **`Track_*.wav` (位於 `vault_ready_for_mix/`)**
   - 經過母帶處理的各單曲無損音檔，供 CEO 手動上傳至串流平台。

3. **`YouTube_CheatSheet_[時間戳].txt`**
   - 自動內聯合體的最終版發行文案（由 Gemini 2.5 Flash 生成）。內含已進行商業化修飾的 Tracklist，CEO 可免編輯直接複製貼上。

## ▌ C. R&S Echoes 產線工作流 (Phantom Matrix Pipeline v15.3)

### Phase 1 — 單一樞紐與背景供彈 (Streamlit Hub & Background Genesis)
**戰術**：
- **背景供彈**：CEO 於 Streamlit 點擊「一鍵供彈」，觸發背景執行緒 (`daemon=True`)。系統呼叫 Gemini 2.5 Flash 產生高質量 Suno 提示詞，全程 UI 零阻塞。
- **拖曳分類入庫**：CEO 於網頁端將生成的滿意音軌拖曳上傳，後端自動分類至 `ceo_approved_beats/{channel}/` 進行頻道隔離。

### Phase 2 — 嚴格母帶處理 (Strict Auto-Mastering)
**腳本**：`audio_mastering_engine.py`
**戰術**：
- **動態發行規格**：使用 FFmpeg `-af loudnorm`。Lofi 鎖定 `-16.0 LUFS`，Light Music 鎖定 `-18.0 LUFS`。
- **自動註冊金庫**：轉換為 44.1kHz/16-bit 無損 WAV 後，存入活水金庫並註冊衍生次數為 0。

### Phase 3 — 金字塔視覺抽樣 (Visual Pyramid Vault)
**腳本**：`visual_vault_db.py`
**戰術**：
- **50/25/25 智能抽樣**：啟動大矩陣時，嚴格依循 50%新片、25%二手、25%三手的比例抽取視覺素材，極大化 API 資產壽命。
- **標籤防重疊**：確保連續場景的 `scene_tags` 絕無交集，避免視覺疲勞。

### Phase 4 — 極速矩陣與預混音 (Modular Rendering & Acrossfade Premix)
**腳本**：`multi_scene_processor.py` (The Core Engine)
**戰術**：
- **Stage 1 (預烤三版本)**：將 8 秒短片切分並加入 `fade` 濾鏡，透過 `concat demuxer` 極速無損放大為 10 分鐘實體影片（自帶呼吸燈轉場）。
- **Stage 2.5 (預混音防斷層)**：將打亂的多首音軌透過 `acrossfade` 縫合成單一的 `audio_premix.wav`，徹底解決時間軸漂移。
- **Stage 2 (終極陣列拼接)**：使用 `-c:v copy` 將 6 個 10 分鐘影片與超長預混音瞬間物理縫合，並用 `-t 3600` 強制鎖死時長。
- **延遲精準扣款**：產出成功後，才對視覺素材執行 `derivation_count + 1` 扣款，達成 100% 資產保鮮。

### Phase 5 — 企劃合體與審計代謝 (CheatSheet & Metabolism)
**戰術**：
- **一站式企劃合體**：Gemini 2.5 Flash 動態產生 SEO 描述與中繼資料，合併 Tracklist 輸出最終版 `YouTube_CheatSheet.txt`。
- **UI 雙重防護審計**：CEO 透過 Streamlit 查閱「資產審計儀表板」。所有的舊檔清理、靶場重置與手動封存，必須在 UI 下方勾選「雙重確認 (Double-Opt-In)」方可執行，實現最高級別的防呆機制。

### E1. 數位資產與財務防禦 (Zero-Risk Distribution Rule)
- **廢除高風險 RPA 自動上傳**：為 100% 規避 Cloudflare 反爬蟲機制與保護版稅帳號，系統已全面淘汰幽靈機器人代操。發行作業回歸「一站式商業企劃合體」，CEO 僅需手動複製貼上。

### E2. 視覺母帶保護 (Static Camera Rule)
- 在呼叫任何 AI 生成微動態影片素材 (I2V) 時，運鏡參數必須強制設定為「靜止 (Static)」。嚴禁任何鏡頭推進或平移，確保首尾無縫循環無破綻。

### E3. 基礎穩定性防線 (Infrastructure Defenses)
- **原子寫入 (Atomic Writes)**：所有的 JSON 企劃書 (`metadata_distrokid.json`) 與資料庫狀態更新，必須使用原子寫入 (`.tmp` → `os.replace()` 或 DB Commit)。嚴防斷電或非同步操作衝突導致的 0KB 靜默損毀。
- **指令注入與 FFmpeg 語法鐵律**：FFmpeg 與 FFprobe 的調用嚴禁使用 `shell=True`，必須以陣列形式傳遞參數。此外，FFmpeg 濾波圖中的數學表達式**必須強制使用單引號 `'` 包覆**，防止底層解析崩潰。

### E4. 零容忍靜默失敗與延遲扣款 (No Silent Failure & Delayed Commit)
- **嚴禁靜默失敗**：程式碼中嚴禁使用空 `try-except` 或僅印出警告卻繼續執行的邏輯。若核心流程 (如 FFmpeg 縫合) 發生異常，必須強制呼叫 `sys.exit(1)` 並將錯誤拋出至 Streamlit UI 報警。寧可產線停擺，也絕不允許產生「空資產」或「假成功」。
- **延遲精準扣款 (Delayed Commit)**：系統必須誠實且精準地反映資產的使用狀態。在選曲或視覺抽樣階段【絕對禁止提前扣款】，必須在成片成功寫入磁碟後，才執行 `derivation_count + 1`，達成 100% 資產保鮮。

### E5. 資產保鮮與透明度防線 (Vault Metabolism & Audit)
- **Streamlit 資產審計儀表板**：UI 內建雙金庫儀表板，即時撈取 SQLite 資訊，提供「待命部隊清單」與「冷金庫水位」，讓 CEO 在網頁端精準掌握供彈與代謝需求。
- **雙重防護代謝授權 (Double-Opt-In)**：嚴格禁止系統擅自刪除 CEO 放入的原始資產。清理靶場與轉移至冷宮的破壞性動作，全權交由 Streamlit UI 下方的「雙重勾選確認」機制把關，徹底根絕檔案遺失風險。
