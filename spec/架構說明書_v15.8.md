## ▌ 1. 技術棧全覽 (v15.8 對齊版)

### 1.1 大腦協同 — 三引擎 LLM 架構

```
┌─────────────────────────────────────────────────────────┐
│               llm_client.py  統一路由閘道                 │
│  provider="minimax" ──▶ MiniMax M2.7 230B (NVIDIA NIM)  │  ← 預設，複雜推理
│  provider="zhipu"   ──▶ 智譜 GLM-4                      │  ← 快速批量生成
│  provider="gemini"  ──▶ Google Gemini 2.5 Flash          │  ← 視覺解構備用
└─────────────────────────────────────────────────────────┘
```

| 引擎 | 模型 | API 協定 | 速度 | 適用場景 |
|------|------|---------|------|---------|
| MiniMax M2.7 (預設) | `minimaxai/minimax-m2.7` | OpenAI-compatible (NVIDIA NIM) | ~100s/次 | 複雜 Agent、發行文案 |
| 智譜 GLM-4 | `glm-4` | ZhipuAI SDK | ~5-10s/次 | 批量提示詞、快速回應 |
| Gemini 2.5 Flash | `gemini-2.5-flash` | google-generativeai | ~15-30s/次 | 視覺解構、多模態 |

> **v15.7 LLM 修復**：MiniMax `max_tokens` 從 2048 → 4096；`_clean_json_response()` 三重加固；`track_count` 上限 20 首防截斷。

### 1.2 動態影音處理引擎（v15.7 廣播級標準）

- **FFmpeg / FFprobe**：所有影音操作的底層引擎
  - `Encode-Once-Repeat`：短素材標準化 1 次後以 `concat demuxer` 無損複製延長
  - `Pre-baked Fade`：素材鑄造 in/mid/out 三版本，物理拼接實現呼吸燈轉場
  - **`Pre-baked Reverse`（v15.7 新增）**：`ping_` 素材額外鑄造 `base_mid_rev.ts`，供 Ping-Pong 交替使用
  - `Closed GOP`：`-flags +cgop -g 48 -keyint_min 48 -sc_threshold 0`，廣播級無縫拼接標準
  - `PTS 歸零優先`：`setpts=PTS-STARTPTS,fps=24` 永遠在 fade 濾鏡之前執行
  - `Acrossfade Premix`：Stage 2.5 將多首音軌預混音為單一長檔，防 EOF 截斷
  - **`v15.8 雙軌 Profile`**：`env_manager.py` 動態載入；Windows → `libx264 CRF29 slow`；Mac M4 → `hevc_videotoolbox q:v 55 hvc1`
  - **音頻碼率（v15.8 統一）**：Stage 2 AAC `-b:a 192k`（前為 320k，聽覺透明已足夠）

### 1.3 數位資產代謝系統

- **SQLite3**：雙金庫並行，`threading.local()` 確保跨執行緒安全
  - `assets/data/rs_music_vault.db`：音樂資產管理（衍生次數、頻道標籤）**v15.8 集中至 assets/data/**
  - `assets/data/veo_visual_vault.db`：視覺素材管理（50/25/25 金字塔抽樣）**v15.8 集中至 assets/data/**
  - `env_manager.py` 提供 `config.music_db_path` / `config.visual_db_path`，全系統統一引用
- **金字塔抽樣**：50% 全新 / 25% 二手 / 25% 三手 + 三層兜底策略
- **Clip Cycling**：當金庫素材不足但非零時，自動循環複用以避免 Fatal Error

### 1.4 智能迴圈策略模組（v15.7 新增）

```
video_loop_classifier.py
  ├── PREFIX_PINGPONG = "ping_"
  ├── PREFIX_FORWARD  = "ford_"
  ├── get_loop_strategy(video_path) → "pingpong" | "forward"
  ├── get_base_name(video_path)     → 去前綴的原始詞幹
  └── classify_vault(vault_dir)     → {"pingpong": [...], "forward": [...]}
```

**Ping-Pong vs Forward 選擇原則：**

| 場景類型 | 策略 | 原因 |
|---------|------|------|
| 海浪、雲、火焰、雨、水流 | `ping_` (Ping-Pong) | 振盪運動，反向播放自然流暢 |
| 人物、lofi 女孩、有人臉的場景 | `ford_` (Forward) | 反向眨眼/說話會觸發恐怖谷效應 |
| 建築內景（無人）、山景、庭院 | `ford_` (Forward) | 已是完美無縫素材 |
| CEO 判斷後手動命名 | 依前綴執行 | 100% 準確率保證 |
