## ▌ 1. 技術棧全覽（v15.11 Windows 對齊版）

### 1.1 大腦協同 — 三引擎 LLM 架構

```
┌─────────────────────────────────────────────────────────┐
│               llm_client.py  統一路由閘道                 │
│  provider="minimax" ──▶ MiniMax M2.7 230B (NVIDIA NIM)  │  ← 預設，複雜推理
│  provider="zhipu"   ──▶ 智譜 GLM-4                      │  ← 快速批量生成
│  provider="gemini"  ──▶ Google Gemini 2.5 Flash          │  ← 視覺解構備用
└─────────────────────────────────────────────────────────┘
```

| 引擎 | 模型 | API 協定 | 速度 | 適用場景 |
|------|------|---------|------|---------|
| MiniMax M2.7（預設）| `minimaxai/minimax-m2.7` | OpenAI-compatible (NVIDIA NIM) | ~100s/次 | 複雜 Agent、發行文案 |
| 智譜 GLM-4 | `glm-4` | ZhipuAI SDK | ~5-10s/次 | 批量提示詞、快速回應 |
| Gemini 2.5 Flash | `gemini-2.5-flash` | google-generativeai | ~15-30s/次 | 視覺解構、多模態 |

**v15.11 LLM 安全規則**：
- `llm_client.py` 透過 `json_parser_utils.parse_llm_json_response()` 統一清洗回傳 JSON（廢除舊版 `_clean_json_response()`）。
- 非同步 LLM 呼叫 `llm_async.py`（`scripts/common/llm_async.py`）：異步批量呼叫介面，支援跨引擎降級 `generate_with_full_fallback()`；重試狀態持久化至 `assets/data/llm_retry_queue.db`。
- MiniMax `max_tokens = 4096`；`track_count` 上限 20 首防截斷。
- TTAPI_KEY 透過 `secrets_manager.get_secrets()` 取得，禁止裸 `os.getenv` 後直接 log。

### 1.2 動態影音處理引擎（廣播級 v15.7+ 標準）

- **FFmpeg / FFprobe**：所有影音操作底層引擎
  - `Encode-Once-Repeat`：短素材標準化 1 次後以 `concat demuxer` 無損複製延長
  - `Pre-baked Fade`：素材鑄造 in/mid/out 三版本，物理拼接實現呼吸燈轉場
  - `Pre-baked Reverse`（v15.7）：`ping_` 素材額外鑄造 `base_mid_rev.ts`，供 Ping-Pong 交替使用
  - `Closed GOP`：`-flags +cgop -g 48 -keyint_min 48 -sc_threshold 0`，廣播級無縫拼接標準
  - `PTS 歸零優先`：`setpts=PTS-STARTPTS,fps=24` 永遠在 fade 濾鏡之前執行
  - `Acrossfade Premix`：Stage 2.5 將多首音軌預混音為單一長檔，防 EOF 截斷
  - **Windows 編碼 Profile**：`libx264 CRF29 slow -tune film`（`config.video_enc_profile["common_enc"]`）
  - **音頻碼率（v15.8 統一）**：Stage 2 AAC `-b:a 192k -ar 48000`（`config.video_enc_profile["audio_enc"]`）

### 1.3 數位資產代謝系統

- **SQLite3**：雙金庫並行，`threading.local()` 確保跨執行緒安全
  - `assets/data/rs_music_vault.db`：音樂資產管理（衍生次數、頻道標籤）
  - `assets/data/veo_visual_vault.db`：視覺素材管理（50/25/25 金字塔抽樣）
  - `env_manager.py` 提供 `config.music_db_path` / `config.visual_db_path`，全系統統一引用
  - **v15.11**：`VaultDatabase._initialize_database()` 加入 `_first_init` 防護，只在首次建立 schema 時印初始化訊息，每次重複實例化（如 UI 渲染、心跳）不再污染 log。
- **金字塔抽樣**：50% 全新 / 25% 二手 / 25% 三手 + 三層兜底策略
- **Clip Cycling**：當金庫素材不足但非零時，自動循環複用以避免 Fatal Error

def _switch_container_gene_pool(self, gene_pool_relative: str) -> Tuple[bool, str]:
    """將指定子風格基因庫內容覆寫到 GL_4M_Suno_prompt.md 容器。"""

    except Exception as e:
